Remarks

Ordinarily, DIType serves to provide the correct debug information type. However, when using the pointer+alloca+memcpy pattern to pass by value the actual source and debug info type is different than the LLVM function signature. This class is used to construct signatures to allow for overriding the normal 1:1 type mapping with DIType.

Since DebugTypePair implements ITypeRef it can be implicitly passed where an ITypeRef is expected. This allows for function signature factories to accept ITypeRef. Then, when explicitly needed, callers can pass in an appropriate DebugTypePair instance.
